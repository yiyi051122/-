基于农业知识增强的小麦病害辅助诊断方法研究 核心技术性内容提取
一、核心技术架构
本研究核心架构为「知识图谱+检索增强生成（RAG）」，整合三大检索模块（知识图谱检索、向量检索、BM25检索），通过RRF融合算法实现多路结果融合，结合贝叶斯优化确定最优融合权重，依托DeepSeek大语言模型完成智能诊断问答，系统前端用Streamlit搭建，后端基于Python和LangChain框架，集成Neo4j、ChromaDB等组件，实现从文本输入到诊断输出的全流程闭环。
二、关键技术及实现细节
2.1 检索技术（三大核心检索模块）
2.1.1 知识图谱检索技术
1. 知识图谱结构：包含8类实体节点（病害、症状、病因、药剂、发病条件、防治方法、传播途径、发病时期、危害部位）和7类关系边，具体关系如下：
•	HAS_SYMPTOM（病害→症状）
•	HAS_CONDITION（病害→发病条件）
•	HAS_CONTROL（病害→防治方法）
•	USES_PESTICIDE（病害→药剂）
•	HAS_TRANSMISSION（病害→传播途径）
•	HAS_STAGE（病害→发病时期）
•	AFFECTS_PART（病害→危害部位）
2. 存储工具：Neo4j 5.12.0 Community Edition，通过Cypher查询语言实现图遍历和模式匹配。
3. 知识抽取方法：规则匹配与词典匹配结合，构建覆盖37种病害、典型症状、常用药剂的领域词典，用Jieba分词识别实体，通过句法模式（如“XX的症状为XX”“XX由XX引起”）捕捉实体关系。
4. 检索流程：
•	步骤1：关键词提取与分类（Jieba分词，剔除停用词，分为病害关键词、症状关键词）；
•	步骤2：多策略分层检索（按匹配方式赋予不同优先级分数）：
        
￮	病害名称匹配检索：MATCH (d:Disease) WHERE d.name CONTAINS $keyword RETURN d.name，优先级10分，最多返回5个实体；
￮	症状关联检索：MATCH (d:Disease)-[:HAS_SYMPTOM]->(s:Symptom) WHERE s.desc CONTAINS $keyword RETURN DISTINCT d.name，优先级5分；
￮	病因关联检索：MATCH (d:Disease)-[:CAUSED_BY]->(c:Cause) WHERE c.desc CONTAINS $keyword RETURN DISTINCT d.name，优先级3分；
￮	药剂关联检索：MATCH (d:Disease)-[:USES_PESTICIDE]->(p:Pesticide) WHERE p.name CONTAINS $keyword RETURN DISTINCT d.name，优先级4分；
•	步骤3：病害-文档映射（三级策略：精确匹配→编辑距离模糊匹配（阈值0.6）→同义词映射字典转换），权重归一化（除以10）后返回Top-K结果。
2.1.2 向量检索技术
1. 核心工具：ChromaDB 0.4.22（向量数据库）、Sentence Transformers 2.2.2（预训练模型：moka-ai/m3e-base，BERT架构中文优化版）。
2. 文档向量化流程：
•	文档加载：读取40多种小麦病害的TXT格式知识文档；
•	文本预处理：多级正则过滤（剔除HTML标签、期刊元信息、参考文献等噪声），统一UTF-8编码；
•	文本分块：递归字符分割，基本块500字符，相邻块重叠50字符，按分隔符优先级（双换行→单换行→句号→分号→逗号→空格）切分；
•	向量编码：模型输出768维稠密向量，进行L2归一化处理；
•	向量存储：存入ChromaDB，自动构建HNSW索引，支持近似最近邻搜索（复杂度O(log n)）。
3. 查询检索流程：
•	查询预处理：与文档预处理一致，保证表示空间统一；
•	查询向量化：使用同一moka-ai/m3e-base模型编码为768维向量；
•	相似度计算：ChromaDB默认采用L2距离，转换为相似度分数，公式如下：
相似度范围[0, 1]，值越大语义越相似，检索返回Top-K文档片段。
￮	L2距离公式：d(q, d) = sqrt( Σ(i=1到n) (q_i - d_i)^2 ) （公式4）
￮	相似度分数公式：similarity = 1 - d(q, d) （公式5）
•	文档映射：三级策略（元数据source字段定位→文档片段前150字符反向匹配→字符重叠度匹配），按相似度降序输出。
2.1.3 BM25检索技术
1. 核心工具：rank_bm25 0.2.2，采用BM25Okapi变体。
2. 倒排索引构建：
•	文档加载：与向量检索共用同一套TXT知识库；
•	分词处理：Jieba分词，过滤长度为1的单字词；
•	词频统计：构建词频字典，记录每个词的文档出现次数、单文档词频、所有文档平均长度；
•	索引构建：建立词元到文档ID及词频的倒排字典，支持O(1)查找。
3. 评分计算：
•	核心评分公式：score(D, Q) = Σ IDF(q_i) * [ f(q_i, D) * (k1 + 1) / ( f(q_i, D) + k1 * (1 - b + b * |D| / avgdl) ) ] （公式6）
•	IDF（逆文档频率）公式：IDF(q_i) = log( (N - n(q_i) + 0.5) / (n(q_i) + 0.5) ) （公式7）
•	变量含义：
        
￮	f(q_i, D)：查询词q_i在文档D中的词频；
￮	|D|：文档D的长度（词元数量）；
￮	avgdl：平均文档长度；
￮	k1：词频饱和度参数（本系统设为1.5）；
￮	b：文档长度归一化参数（本系统设为0.75）；
￮	N：文档总数；
￮	n(q_i)：包含查询词q_i的文档数量。
4. 检索流程：用户查询分词过滤→计算查询词IDF→遍历含查询词文档累加BM25得分→按得分降序返回Top-K结果（仅保留得分>0的文档）。
2.2 多路检索融合技术
2.2.1 倒数排名融合（RRF）
1. 核心公式：
•	RRF核心公式：RRF_score(d) = Σ w_i / (k + rank_i(d)) （公式8）
•	单路RRF分数公式：rrf_score_i = w_i / (k + rank_i) （公式9）
•	变量含义：
        
￮	d：待融合的文档；
￮	m：检索模块数量（本系统m=3）；
￮	w_i：第i个检索系统的权重；
￮	rank_i(d)：文档d在第i个检索系统中的排名（从1开始）；
￮	k：平滑常数（本系统设为60）。
2. 融合流程：
•	步骤1：初始化空字典，用于存储每个文档的累计RRF分数；
•	步骤2：遍历每个启用的检索模块，提取其Top-10结果，计算单路RRF分数（模块权重/(k+rank)），存入临时列表；
•	步骤3：按文档名聚合累加分数（首次出现初始化分数为0，后续直接累加）；
•	步骤4：按累计RRF分数降序排列（分数相同按文档名升序），输出Top-K（默认K=5）结果。
2.2.2 贝叶斯优化（权重优化策略）
1. 优化问题定义：
•	优化公式：w* = argmax( w∈Ω ) F(w) （公式10）
•	变量含义：
        
￮	w = (w_kg, w_dense, w_bm25)：权重向量（分别为知识图谱、向量检索、BM25的权重）；
￮	Ω = [0.1, 5.0]^3：权重搜索空间（下限0.1避免模块被忽视，上限5.0避免单一模块主导）；
￮	F(w)：权重配置w下的F1分数（目标函数）。
2. 优化流程：
•	步骤1：划定权重搜索范围[0.1, 5.0]^3；
•	步骤2：定义目标函数（加载700个问答对验证集，用当前权重跑RRF融合检索，计算F1分数）；
•	步骤3：采用高斯过程作为代理模型，采集函数选用期望改进（EI），迭代50次（每次迭代：采集函数选权重点→实测F1分数→更新代理模型→记录最优权重）；
•	步骤4：选取F1分数最高的权重组合作为最优解，保留优化历史。
3. 优化结果：最优权重配置为w_kg=4.785（知识图谱）、w_dense=0.754（向量检索）、w_bm25=4.182（BM25）；优化后精确率从63.93%提升至72.03%，F1分数从72.19%提升至79.65%，召回率和命中率稳定在接近100%。
2.3 评估指标
采用4个核心评估指标，均按病害名称口径统计：
•	1. 召回率（Recall）：检索出的相关文档数 / 所有相关文档总数 （公式1），衡量“找得全不全”；
•	2. 精确率（Precision）：检索出的相关文档数 / 检索出的文档总数 （公式2），衡量“找得准不准”；
•	3. F1分数：精确率和召回率的调和平均，公式：F1 = 2 * (Precision * Recall) / (Precision + Recall) （公式3），作为主要评估依据和贝叶斯优化目标函数；
•	4. 命中率（Hit Rate）：命中相关文档的查询数 / 总查询数，衡量系统基本可用性。

四、关键技术补充说明
•	文本预处理：采用多级正则过滤噪声，递归字符分割（500字符块+50字符重叠），确保语义完整性；
•	向量归一化：所有文档向量和查询向量均做L2归一化，使欧氏距离可直接反映语义相似度；
•	检索优先级：知识图谱检索按匹配方式赋予不同分数，避免单一实体过度占优（同一病害仅保留首次命中分数）；
•	融合优势：三模块融合实现100%召回率和78.56% F1分数，显著优于单模块，体现结构化知识与非结构化文本的互补性。
